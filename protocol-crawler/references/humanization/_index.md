# 拟人化交互与反检测要点

## 核心原则

> 目标不是"唯一"，而是"平庸"。你的zw越普通、与大众越撞车，你越安全。不要让自己变得特别，要让自己消失在人群中。

## 文件结构

### pc 直接相关（必读）

| 文件                  | 内容                                                                                    | 何时使用                                             |
| --------------------- | --------------------------------------------------------------------------------------- | ---------------------------------------------------- |
| `network-layer.md`    | TLS zw（JA3/JA4）+ curl_cffi 配置、HTTP/2 头部帧顺序、资源加载瀑布流、缓存与 Cookie Jar | **所有 pc 项目都应参考**，尤其被 TLS 检测/403 拦截时 |
| `proxy-management.md` | dl 类型选择、存活校验、轮换策略、失败淘汰机制                                           | 需要 dl 轮换 / 被 IP 封禁时                          |

### 浏览器辅助场景（按需参考）

> ⚠️ 以下文件仅在使用浏览器自动化（如登录态获取、yzm 参数分析）时参考。
> pc 数据采集层禁止使用浏览器自动化。

| 文件                                        | 内容                                                 | 何时使用                       |
| ------------------------------------------- | ---------------------------------------------------- | ------------------------------ |
| `browser-automation/browser-fingerprint.md` | 浏览器 zw（WebGL/字体/Canvas/视口）检测与对抗        | 浏览器辅助场景被 zw 检测拦截时 |
| `browser-automation/mouse-biometrics.md`    | 鼠标轨迹生物特征模拟（贝塞尔 + 柏林噪声 + 菲茨定律） | 浏览器辅助场景需要模拟鼠标时   |
| `browser-automation/keystroke-dynamics.md`  | 键盘输入节奏模拟（N-Gram 延迟 + Key Rollover）       | 浏览器辅助场景需要模拟键盘时   |
| `browser-automation/scroll-navigation.md`   | 滚动与导航行为拟人化（惯性滚动 + 阅读节奏）          | 浏览器辅助场景需要模拟滚动时   |

## 技术栈推荐

| 层次                   | 推荐工具                                                           |
| ---------------------- | ------------------------------------------------------------------ |
| 网络层（pc 必备）      | `curl_cffi`（TLS zw 模拟）                                         |
| 浏览器控制（辅助场景） | `Playwright` + stealth 插件（仅用户建议/要求时，禁止用于数据采集） |
| 轨迹生成               | `py-ghost-cursor` / `GhostCursor`                                  |
| 数学库                 | `numpy`, `scipy`, `noise` (Perlin)                                 |
